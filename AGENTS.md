# 项目协作规则

以下规则适用于本目录及全部子目录，优先级高于 Agent 的默认工作习惯。

## 修改前必须核实并确认

1. 收到“检查、看看、审查、核对、诊断、解释”等请求时，默认只读；不得修改任何代码、文档、配置或生成文件。
2. 任何修改开始前，必须先查阅并核实相关资料。涉及 Amazon API 时，优先拉取或查阅 Amazon 官方最新文档、官方 OpenAPI 模型或官方源码；不得仅凭记忆、搜索摘要或非官方文章下结论。
3. 核实后必须先向用户说明：
   - 发现了什么问题；
   - 资料依据是什么；
   - 准备修改哪些文件；
   - 每个文件准备改什么；
   - 可能的兼容性、数据流和边界影响；
   - 准备运行哪些测试。
4. 完成上述说明后，必须等待用户明确同意修改。没有得到明确同意前，不得调用文件编辑工具，也不得运行会改写文件的构建、格式化、代码生成或安装命令。
5. 用户同意后，只能修改已说明并获准的范围。若过程中发现需要扩大范围，必须停止并再次征得用户确认。

## 中断与继续

1. 用户在执行过程中发来新消息或中断操作时，立即停止后续修改。
2. 先报告已经完成的动作、已修改的文件和尚未执行的步骤。
3. 必须询问用户是否继续；未得到明确答复前不得恢复执行。

## 修改与验证

1. 修改时注意跨文件一致性、参数和数据流完整性、错误契约、兼容性以及边界条件。
2. 测试应覆盖正常路径、错误路径、边界值和修改涉及的数据流；只运行与获准范围相关的验证。
3. 不得调用真实 Amazon 写 API，除非用户单独明确授权。
4. 完成后如实报告修改内容、测试结果、未验证事项和剩余风险；不得把未执行的测试描述为通过。
